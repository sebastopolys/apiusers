ENDPOINT

The default Endpoint is named like the plugin name: apiusers

There is no link supplied on the frontend, just access by typing the endpoint after website URL.

Or place a link anywere on the frontend https://yourwebsite.com/apiusers

COMPOSER

Composer is integrated to the proyect and a composer.json file is available

I already had some experience using composer with Laravel, but never did it with WordPress before, so this was my first time using composer with WP

Pugin can be installed on a WP website following these steps:

  1. Open console and cd to the plugins path of WP: https://yourwebsite.com/wp-content/plugins
  2. Run git clone https://github.com/sebastopolys/apiusers/
  3. cd to the apiusers folder https://yourwebsite.com/wp-content/plugins/apiusers
  4. Run composer install
  5. Run composer update if necesary

BACKEND OPTIONS

Some backend options are provided on the WP admin dashboard / Api Users / Settings

  • Customize endpoint: It allows to edit the default endpoint
  • View: Displays table integrated with theme or hiding header & footer. This is done with a simple CSS script
  • Credits: Nothing special, just the option to hide / display a brief footer message
